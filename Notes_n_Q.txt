Ferrite.IdentityMapping:
Notes on SimpleCellValues Function

Purpose:
  - Creates a SimpleCellValues object for a single finite element.
  - Precomputes all shape function values, gradients, and quadrature information.
  - Acts as a data container to make FEM assembly efficient.

Inputs:
  - qr::QuadratureRule     : Quadrature rule (points and weights) for integration.
  - ip_fun::Interpolation  : Interpolation object for the solution field.
  - ip_geo::Interpolation  : Interpolation object for the element geometry.

Steps:

1. Determine spatial dimension:
   - dim = Ferrite.getrefdim(ip_fun)
   - Gets the reference dimension (1D, 2D, 3D) of the element.

2. Get quadrature points and weights:
   - weights = Ferrite.getweights(qr)
   - n_qpoints = length(weights)
   - T = eltype(weights) (data type, usually Float64)

3. Allocate arrays for function interpolation:
   - n_func_basefuncs = getnbasefunctions(ip_fun)
   - N = zeros(T, n_func_basefuncs, n_qpoints)       # Shape function values
   - dNdx = zeros(Vec{dim,T}, n_func_basefuncs, n_qpoints)  # Physical gradients (to be computed later)
   - dNdξ = zeros(Vec{dim,T}, n_func_basefuncs, n_qpoints)  # Reference gradients

4. Allocate arrays for geometric interpolation:
   - n_geom_basefuncs = getnbasefunctions(ip_geo)
   - M = zeros(T, n_geom_basefuncs, n_qpoints)       # Geometric shape values
   - dMdξ = zeros(Vec{dim,T}, n_geom_basefuncs, n_qpoints)  # Geometric reference gradients

5. Precompute shape function values and gradients at quadrature points:
   - Loop over quadrature points:
       for (qp, ξ) in pairs(Ferrite.getpoints(qr))
   - For each solution shape function i:
       - dNdξ[i, qp], N[i, qp] = Ferrite.reference_shape_gradient_and_value(ip_fun, ξ, i)
   - For each geometric shape function j:
       - dMdξ[j, qp], M[j, qp] = Ferrite.reference_shape_gradient_and_value(ip_geo, ξ, j)

6. Initialize detJdV:
   - detJdV = zeros(T, n_qpoints)
   - Will store the physical quadrature weights after reinit! is called.

7. Return:
   - SimpleCellValues(N, dNdξ, dNdx, M, dMdξ, weights, detJdV)

---

Data stored in SimpleCellValues:
  1. Shape function values (N)
  2. Shape function gradients in reference coordinates (dNdξ)
  3. Shape function gradients in physical coordinates (dNdx, initially zero)
  4. Geometric mapping info (M, dMdξ)
  5. Quadrature information (weights, detJdV, initially zero)

Use in FEM:
  - During assembly, loop over elements and quadrature points.
  - Use precomputed N, dNdξ, dNdx, M, dMdξ, and quadrature weights to compute:
      - Stiffness matrices
      - Mass matrices
      - Residual vectors
  - Efficient because all repetitive calculations are precomputed once.

Relation to Ferrite.IdentityMapping:
  - SimpleCellValues works with standard Ferrite types and functions.
  - Can be reinitialized for current cell geometry using reinit! to fill dNdx and detJdV.


=== Tuple Unpacking in Julia ===

1. Definition:
   - Tuple unpacking allows you to assign multiple values returned by a function (or present in a tuple) to multiple variables in a single line.
   - Syntax:
       var1, var2, ..., varN = function_that_returns_tuple(...)

2. Example 1: Simple numbers
   function minmax(x, y)
       return (min(x, y), max(x, y))
   end

   a, b = minmax(10, 25)
   println(a)  # 10
   println(b)  # 25

   - Here, minmax returns a tuple (min, max)
   - a gets first element, b gets second element

3. Example 2: FEM context
   dNdξ[i, qp], N[i, qp] = Ferrite.reference_shape_gradient_and_value(ip_fun, ξ, i)

   - Ferrite.reference_shape_gradient_and_value returns a tuple:
       (gradient_of_shape_function, value_of_shape_function)
   - dNdξ[i, qp] gets gradient
   - N[i, qp] gets value

4. Notes:
   - The number of variables on the left must match the number of elements in the tuple
   - You can give any names you like to the variables
   - Works with any iterable that returns tuples (arrays with indices, dictionaries, etc.)
